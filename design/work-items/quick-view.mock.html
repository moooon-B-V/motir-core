Email + password sign-in

Blocked Waiting on 2 work items — PROD-3, PROD-8
Description

Let a returning user sign in with their email address and password. Validate the credentials against the stored argon2id hash, establish a session, and route the user to the workspace they last visited.

Surface a single generic error for any wrong email-or-password combination — never reveal which field was wrong — and rate-limit repeated failures per account to blunt credential-stuffing. After five failed attempts within fifteen minutes, add an increasing backoff before the next attempt is accepted.

The flow must handle these cases:

  • Unverified email → block sign-in and offer to resend the verification link.
  • Account created via Google OAuth with no password set → prompt to use “Continue with Google”, not a password.
  • A pending workspace invite for the signing-in email → after auth, route into the invite-accept flow rather than the default workspace.

On success, set the session cookie, record last_signed_in_at, and redirect. The form preserves the typed email across a failed attempt so the user only re-enters the password.

Explanation, child items, the full relationships / links panel, attachments, and the activity feed live on the full page.
