Work-item quick view (peek)

Story 2.5 · design for 2.5.18 → 2.5.19. A modal that previews a work item without leaving /issues, with a prominent path to the full detail page. Composes the shipped Modal · IssueTypeIcon · Pill · Avatar — no new primitive. The row gains a trailing actions cell + a stretched-link so the quick-view button never nests inside the row link. Built from the real --el-* tokens.

The whole-row navigation link is rendered as a stretched link (an ::after overlay on the relative row) instead of a wrapping <a>. The quick-view <button> lives in a trailing actions cell layered above the overlay (z-index), so button and link are siblings — never nested. The button is hidden at rest, revealed on row :hover / :focus-within, and always shown on touch. One shared row change serves both Tree and List. The peek shows a condensed subset of the 2.4 detail page: type icon + linked PROD-N + Status pill (header), title, the three glanceable meta fields (Assignee · Priority · Due), and a 3-line description excerpt. Reporter, Estimate, parent/children, relationships, labels, full Markdown description/explanation, and the activity feed stay detail-only — reached via the prominent Open full page → button (and the header identifier). Backdrop click, the ×, and Esc all close; the Modal primitive handles focus-trap + return-focus to the triggering row button.

4 · Quick view — not found / no access (stale or forbidden key)

This issue isn’t available

PROD-23 may have been moved, deleted, or it’s in a workspace you don’t have access to.

A stale ?peek= key (deleted issue) or a forbidden one (cross-workspace — finding #44's pure-workspace gate returns the same not-found, never leaking existence) shows this single empty state. The search-x glyph covers not-found; a lock glyph variant covers an explicit no-access message where the product chooses to distinguish them. No Open full page button here — there is no page to open.
