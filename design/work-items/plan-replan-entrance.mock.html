The scoped universal workspace composes the shipped PlanningWorkspace (MOTIR-1193) + WorkItemCanvas (MOTIR-1194) — it does NOT redraw either. Map (left): the canvas is scoped to the item's NEIGHBORHOOD — the item itself (highlighted with an active ring), its parent (ghost), its siblings — using the shipped WorkItemCanvas's presentational contract (forest + edges in, data-driven). Proposed changes render as DASHED nodes with the accent glow (the .proposed style from 7.20.1). Done work is LOCKED — a lock icon, reduced opacity, and a solid border (no interaction). The planner (MOTIR-908) locks done nodes; the confirmation gate (MOTIR-911) rejects any delta that touches a locked node. Chat (right): the shipped chat rail, with the item scope shown in the header. Re-plan mode: the opening turn (the user's reason text) is shown as the first chat bubble. The AI responds with a summary of what it's planning. Plan mode: no opening turn — the AI's first message is "Planning MOTIR-42…" (the "Planning…" state), then "Here's a proposed plan…". Confirm gate (bottom bar): the MOTIR-911 gate renders as a full-width bar at the bottom of the canvas. It shows the count of pending changes, a "Discard" button, and a "Confirm & persist" button. The gate IS the confirm surface — there is NO separate modal or page. Empty delta → bar shows "No changes to apply" and the confirm button is disabled.

The entrance is the SAME affordance for every kind. The Plan / Re-plan label is gated on whether the item has children (Plan = no children, Re-plan = children exist). The global PlanWithAILauncher (MOTIR-1299, the hero pill in TopNav) is ALREADY present on every screen — this contextual button is the per-item entrance that opens the workspace SCOPED to this item's neighborhood, distinct from the global launcher's project-level scope.
