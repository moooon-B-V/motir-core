Group order is fixed: Starred → My filters → Project filters → Defaults (starred entries appear ONLY in Starred — no duplicates). Search is server-backed + debounced over the user-visible set (bounded reads — a project with 500 filters never ships them all; finding #57). Applying an entry resolves the envelope into the builder and writes ?filter=v1: — the URL stays the only state channel. The star is a sibling focusable button inside the option row (never nested in another button); starred = filled amber star + aria-pressed. Defaults are the eight expressible built-ins (My open issues · Reported by me · All / Open / Done issues · Created / Updated / Resolved recently): non-editable, non-deletable, NOT starrable and NOT subscribable — they are non-persisted AST constants with no row to star or FK a subscription to.

Vocabulary. Every element composes shipped primitives: the toolbar ToolbarButton (+ the 2.5.4 active grammar), the 6.1.3 summary-chip row (the name chip prepends it, lavender tint + --el-text-strong — AA per finding #35), Modal (--radius-modal / --shadow-modal), FormField + Input/Textarea, the 6.4.1 access-card radio grammar (visibility), the Combobox menu/option vocabulary (dropdown + row menus + day/time selects), Segmented (frequency), Pill tones (visibility + Built-in), the list-table + 2.5.12 pagination grammar (directory), EmptyState/ErrorState, Avatar, and the ink Tooltip. Colour rides Tier-3 --el-* only; shape rides the element-semantic tokens (--radius-btn/card/input/modal/control/badge, --spacing-control/input/chip/icon-btn/tooltip/btn-*, --height-control/input/btn-*, --shadow-subtle/elevated/modal) so data-display-style reshapes everything. The starred star is the one non-grey/primary accent beyond the tints: filled --el-warning amber + aria-pressed (state never colour-only). Toggle dark for token parity.
