The Type picker is the shipped Combobox (not searchable — a fixed, closed set of ten). Because the create modal is a role="dialog", the menu renders inline (the Combobox in-dialog branch), not portaled.

Each row = the type's saturated --el-type-* glyph (mirroring IssueTypeIcon) + its label; the selected row carries the trailing Check. The trigger shows the chosen type's glyph + label + the ChevronsUpDown affordance.

Leaf-only: Type + Executor render only when the chosen kind is a leaf (task / subtask / bug) — absent for epic / story (panel 2d). type may be left unset (null, panel 2c).

The seven "produces an artifact" types take the seven saturated hues; the three "meta / admin" types (decision, manual, chore) take the three greys — desaturation is semantically apt and needs no new Tier-0 colour. The chip puts the hue in a color-mix tint background with --el-text-strong text (AA, finding #35) — so one saturated --el-type-* token yields both the glyph hue and the chip tint. "Either" types default to coding_agent but flip freely.
