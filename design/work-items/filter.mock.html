PROD-12 / PROD-31 are shown only as context (muted) so the matching bugs keep their place in the tree; they are not themselves matches. A "clear filters" reset (popover header, or the trigger) returns the full unfiltered tree.

Notes. The popover is a Popover (role="dialog") anchored under the enabled [Filter] ToolbarButton. Each facet is a multi-select role="listbox" of role="option" rows reusing the Combobox option vocabulary (leading glyph · label · trailing Check) — kind rows take the IssueTypeIcon hue, status rows the StatusPicker dot (status.color ?? category var), assignee rows the member-combobox Avatar with Unassigned first. Colour flows only through --el-*; the count badge / selected-row tint / Check use --el-accent; AA-safe (finding #35 — hue in the tint, strong text). Filters apply live and serialize to the URL — there is no Apply button.
