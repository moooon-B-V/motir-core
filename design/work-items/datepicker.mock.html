DatePicker calendar component

Story 2.4 · design for 2.4.12 (components/ui/DatePicker) — the design-system replacement for the native <input type="date"> calendar. Built from the real tokens/primitives.

0 · Placement — the issue Due-date field

The trigger lives wherever a Due date is edited — the core-fields rail (2.4.2), the create modal (2.3.3), the edit form (2.3.6). It replaces the native date input's field + popup with the design-system field + the floating calendar below. Other rail fields are unchanged.

1 · Trigger — Input-styled field (placeholder / filled / focused / disabled)

The trigger is an Input-styled field (--height-input 44px, --radius-input, --el-border-strong hairline, --spacing-input-x padding). A leading calendar glyph; the value formats via formatDate ("Jun 4, 2026"); a trailing Clear × shows only when a date is set (Due date is nullable). Focus gets the standard 2px primary ring.

2 · Open — the calendar popover (anchored under the trigger)

The calendar floats in a Popover (--radius-card, --shadow-elevated, --el-border), anchored to the trigger. Caption + prev/next month nav, the weekday header, a 6×7 day grid, and a footer with Today + Clear. Selected = Jun 4 (filled); today = Jun 5 (ring + dot). The picker below is live — click days, use the nav, or focus a day and use the arrow keys.

← → ↑ ↓ day by ±1 / ±7 · PageUp/PageDown prev/next month · Home/End week start/end · Enter/Space select · Esc close (focus returns to the trigger).

3 · Day-cell states (the authority for 2.4.12)

Every state routes through --el-*. Today and selection are not conveyed by colour alone (WCAG 1.4.1 / finding #35): selected is a filled circle + bold, today is a ring + a dot under the number — both differ in shape, not just hue. Out-of-month days are faint but navigable; disabled (out of min/max range) are faint + struck-through and not selectable.

4 · Cleared — Clear / Esc returns to the placeholder

Pressing Clear (trigger × or the footer link) sets the value to null and the field returns to its placeholder; the × disappears. The Due date is nullable, so this is a first-class state — not an error.
