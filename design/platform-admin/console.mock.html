Platform admin console — access path · nav shell · estate · usage/cost · drill-down

Epic 10 · Story 10.1 · Subtask 10.1.1 design asset — built from app/globals.css tokens. An internal operator console for Motir platform staff (moooon B.V.), gated to /admin and a 404 for everyone else. Panel 1 draws how staff enter it; Panels 2–5 render inside the console’s persistent left-nav shell so every screen shows where you are and how to reach the next. Read-mostly (governance actions are Story 10.3). Mirror = SaaS superadmin panels (Supastarter / MSP dashboards / the Stripe-estate idiom). Source of truth = this HTML; see design-notes.md. Gates 10.1.4 / 10.1.5 / 10.1.6 and seeds the shared shell for Stories 10.2 + 10.3.

1 · Access path — how a platform-staff user ENTERS the console. The normal-app account menu carries a staff-only “Platform admin” item → /admin. For a non-staff user the item is absent AND /admin 404s (the surface does not exist for them; see Panel 6a). Show the door, not just the room.

The account menu, open. A platform-staff account (e.g. ops@moooon.net) sees an extra “Platform admin” item — the single door into the operator console at /admin. Selecting it opens the shell in Panels 2–5.

For a normal user the item is simply absent, and typing /admin directly returns the standard app 404 (Panel 6a) — the route’s very existence isn’t revealed. The staff gate is a net-new capability; see design-notes.

Selecting “Platform admin” lands on the console Overview (breadcrumb Platform · Overview), with the left-nav shell below.
