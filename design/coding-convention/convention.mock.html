Version history

An audit runs on three triggers: code changes (on demand or a repo webhook — proposes a new convention version if the code drifts), and convention changes (approving an edited convention re-audits existing code against the new standard). The approved standard is retained and never silently overwritten — a new version must be re-approved to supersede.
