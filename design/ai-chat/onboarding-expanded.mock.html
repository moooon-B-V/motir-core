Runtime contract — both axes resolve on <html>: data-display-style flips the shape tokens (Tier-2), data-palette flips the --el-* element tokens (Tier-3). Components reference --el-* and the semantic shape tokens ONLY, so neither axis touches component code — exactly the existing two-tier swap. The wizard writes both attributes; the registry (7.3.32) lists the styles + palettes the gallery / palette steps render.
Feeds 7.3.32 — this panel IS the style schema: the dimensions a style controls (left) and a palette controls (right), each mapped to the exact token roles in app/globals.css, plus the registry of what's shipped vs. "Soon".
