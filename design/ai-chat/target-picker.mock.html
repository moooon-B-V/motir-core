The tray holds the target SET. Picking a work item from the @ dropdown inserts its chip into the targets tray; the input text stays focused so the user can keep typing. The ⨉ button on each chip removes that target from the set (no confirmation). The chips REUSE the shipped WorkItemRefChip vocabulary (internal-links.mock.html panels 0–2: .wi-chip, type icon · mono key · title — no status dot in this context, because a target assignment doesn't imply status relevance). The @ button (panel d) is a visible trigger affordance to the left of the placeholder — clicking it focuses the input and opens the dropdown with the last query, or a "type to search" hint when there's no previous query. The tray label reads "Target" (singular) or "Targets" (plural) depending on count.

States match the shipped work-item @-mention picker convention from internal-links.mock.html panel 3 (short query "keep typing", loading spinner, no-results copy) and comments.mock.html panel 5 (the member mention empty-hint pattern). The difference: the chat dropdown shows ONLY work items (no People section — chat @-mention of members is not in scope for this surface). The empty state appears when the dropdown opens with no typed query (e.g. clicking the @ button); the short-query hint appears when the query is fewer than N characters (N = 2, matching the shipped mention-search debounce threshold).

Pre-scoped from the entrance (MOTIR-1489). When the user activates Plan/Re-plan from a work item's detail page or quick-view modal, the universal planning workspace opens with that item already in the target set — the tray carries its chip, the canvas highlights it with the active ring + "Target" pill, and the AI's opening turn acknowledges the scope. The user CAN remove it (⨉) and/or add more via the @ picker — the entrance sets the initial target, not a locked one. The model is: the entrance passes a target key; the session is seeded with it; the user extends or replaces at will. This maintains consistency with the entrance's Re-plan flow — the reason is the chat turn text, the target is the item the user's operating on.
