(2) The sort / rank tabs — Trending (surging now) · Popular (most-upvoted overall) · New (recently made public). Trending is the DEFAULT landing tab + carries a recency-window selector

Sort & rank

The GitHub-Trending / GitLab-Explore tab set. Each rank is a deterministic total order computed at the read layer over the 6.12.6 vote + activity signals, riding the same keyset cursor — never a precomputed column, never load-all.

Trending window
Trending Default

Surging now. Upvotes + project activity inside the recency window, so a freshly-surging project rises above a higher-lifetime-but-stale one.

Σ(upvotes + activity) within window · recent-weighted
Popular

Most-upvoted overall. Lifetime demand — total upvotes + total viewers across the project's whole public life.

total upvotes + total viewers (lifetime)
New

Recently made public. Ordered by the moment the project's access level turned public (madePublicAt), newest first.

order by madePublicAt desc

The active rank is carried in the URL (?rank=trending&window=week) and composes with the search + tag params.
