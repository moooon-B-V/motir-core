Token scope selection — create-modal toggles + list display

Story 7.7 · Subtask 7.7.18 — extends design/settings/account-settings (7.7.2). Built from the real design system (--el-* colour + element-semantic shape tokens). Reference for the 7.7.19 code subtask.

1 · Create token — the full Modal with the new PERMISSIONS picker, shown UNTRUNCATED so every scope is visible. The 6 scopes (7.7.16) as grouped, plain-language Switch toggles. Default: every scope ON except Delete, which sits in its own rose Danger zone, OFF. Binding scope (workspace) + Expires are unchanged from 7.7.2. In-app the dialog never grows this tall — see Panel 2 for how it fits the viewport.

Default = all on except Delete. A token created without touching the picker can read, edit, manage sprints, archive, and connect integrations — the everyday agent set — but cannot delete. Granting Delete is always a separate, deliberate flip.

2 · How it fits — the shipped Modal caps at max-h-[90vh] and is a flex column, so the fields go in a scrolling Modal.Body while the Cancel / Create footer stays pinned. (Today's CreateTokenModal uses a plain <form>; the scope picker makes it tall enough that 7.7.19 must move the fields into Modal.Body + Modal.Footer — see design-notes.) Never grows off-screen.
