Panel 4. Fills the dashed committed-points slot the backlog reserved (design/backlog/design-notes.md — "filled by Story 4.3"), in the SAME slot position so it drops in with no relayout. Committed (sum of the sprint's points), done (category = 'done' subset, --el-success), and remaining — all bounded grouped aggregates (SUM … GROUP BY with a done predicate), never a client sum of the loaded rows. A planned sprint shows committed only; an unestimated sprint shows —. This is the compact backlog-header figure; the fuller scrum sprint-header summary (Story 4.5) reuses the SAME rollupForSprint aggregate (4.5.2 consumes it for SprintSummaryDto.points).
