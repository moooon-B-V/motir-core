Roadmap — the deterministic work-item canvas

MOTIR-1009 / 7.3.76. One reusable canvas that renders any project's work-item tree + pre-plan phase in a FIXED, opinionated layout — the auto-arranged sibling of the free-form spatial canvas (MOTIR-1235). Rendered on the shipped PlanningCanvas engine (MOTIR-1236: pan / zoom / drag / fit + read-only edges). Reused by start-fresh onboarding (7.3), migrate-existing onboarding (7.15), re-planning (7.11), the persistent roadmap (7.19), and the public project page.

Sheets 1–5 spec the canvas. Sheet 6 specs MULTI-LEVEL navigation: because the planner orders same-level work as a chain at every level (epic↔epic · story↔story · subtask↔subtask — `plan.md`’s dependency-arrow rule), the canvas renders the story-level and subtask-level chains, not just the epic level, via drill-down (decided, Yue 2026-06-22).

DECIDED · drill-down

Multi-level chains — drill-down (self-similar)

The plan is a chain (DAG) at EVERY level (epic↔epic · story↔story · subtask↔subtask — `plan.md`’s dependency-arrow rule), so the canvas renders the story-level and subtask-level chains too. The model is drill-down: click a node and the canvas REFRESHES to that node’s children, laid out as their own chain; a breadcrumb + “Back” walks you up. One level fills the screen, so each chain stays legible at any tree size. Below: the three states as you drill Plan ▸ Invoices ▸ Create invoice.

One level fills the screen at a time — each chain stays big + legible no matter how large the tree (the completeness / scale axis). You drill DOWN into a node’s children and Back UP via the breadcrumb; zoom-to-fit gives a whole-level overview. This EXTENDS the shipped spatial canvas’s “self-similar at every level” — the same node + edge language (the shipped StationCard / PlanningCanvas, --el-type-* hues, neutral firm / dashed-pending edges + the cross-story warning edge) renders at every level; only the per-level node + edge set is re-fed to the engine on drill.
