Roadmap — refresh control & URL-addressable scope · MOTIR-1540

Two usability additions to the shipped roadmap header (Story MOTIR-1539): a header REFRESH control that re-fetches the canvas in place (no full page reload), and making the scope toggle URL-addressable (/roadmap?scope=sprint). This design COMPOSES the shipped RoadmapView header + ProjectRoadmapCanvas (MOTIR-1194) — it does NOT redraw the canvas engine. Only the header refresh control and its states are new pixels.

Idle. The refresh control is a Button (variant secondary, icon-only) carrying a lucide RefreshCw glyph, sat in the right cluster AFTER the scope Segmented toggle. It matches the toggle's 40px outer height so the two align. Accessible name + tooltip: "Refresh roadmap". Clicking it clears the canvas level cache and bumps the canvas reloadKey — an IN-PLACE refetch, so the current drill level, breadcrumb, and zoom/pan are preserved (see panel 2).

Loading. While the refetch is in flight the control swaps its icon for a Spinner, sets disabled + aria-busy="true", and a small "Refreshing…" toast + a faint veil sit over the canvas. Crucially the breadcrumb still reads Epic 7 — the refresh re-loads the CURRENT level in place (reloadKey bump → loadLevel(focusId)), it does NOT remount the canvas to root. The control returns to idle (panel 1) when the fetch resolves — driven by the real fetch-completion signal, never a fixed timer.

4 · URL-addressable scope— the toggle is visually UNCHANGED; scope now lives in the URL /roadmap · /roadmap?scope=sprint

Shareable. Deep-link, reload & browser Back land here.

No new pixels — a behaviour note. The scope Segmented toggle is visually unchanged. What changes: selecting Active sprint writes ?scope=sprint to the URL (router.replace, scroll:false); the default Whole project omits the param (clean /roadmap). The server page seeds the initial scope from the URL, so a deep-linked / reloaded / shared ?scope=sprint opens directly in sprint scope, and browser Back/Forward moves scope. This is the standard view-state URL convention (Linear / Jira / GitHub Projects).

Access path: the refresh control is an in-header affordance on the EXISTING Roadmap view, reached from the shipped Roadmap primary left-nav entry (MOTIR-1011) — no new nav entry. Both additions COMPOSE the shipped roadmap header + canvas; the canvas engine (ProjectRoadmapCanvas / PlanningCanvas) is not redrawn.
