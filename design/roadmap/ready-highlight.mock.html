The rule. A node is ready iff it is not yet started/done (a to-do status) AND every item it is blocked_by is done — the shipped ReadinessDto.ready / dispatch-ready set. The highlight marks the actionable frontier, so it fires on exactly one of the states above. It is deliberately distinct from the accent "you are here" border (in-progress frontier), the red dependency flag, and the status pill — by both hue (success green) and shape (a left accent bar + a pill, not a full border). Holds in both Whole-project and Active-sprint scope.

  • "Ready" pill in the status slot (replacing the dim "To do" pill — a ready item IS a to-do whose blockers are all done, so "Ready" is the informative label). --el-tint-mint background, --el-text-strong text (AA), a CirclePlay glyph in --el-success.
  • Success left accent bar — a 3px --el-success inset bar (box-shadow: inset 3px 0 0, respects --radius-card) so a ready node is scannable across a dense canvas at zoom, without adding a full border that would read like the accent "you are here" frame.
  • Everything else is the shipped WorkItemNode unchanged — kind tile, identifier, title, drill chevron, progress meter (on containers).
