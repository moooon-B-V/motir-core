The decision

A manual/human node carries a compact "Manual" work-type chip in the node's top row (right cluster). An agent node carries nothing new — the roadmap is a dispatch-scanning surface, so the majority (agent-run) nodes stay clean and the human-gated ones POP. This is a justified rung-1 deviation from "show the type chip on every row" (as the denser /ready and list/tree rows do, 8.8.9 / 8.8.10): on a pannable canvas a chip on every node is clutter, and the whole point of the ask is that manual stands out. Trigger: the shipped predicate executor === 'human' || type === 'manual' (lib/dto/ready.ts) — the SAME one the ready-list Show instruction keys on.

Composed, not invented

  • Glyph + hue + tint are the shipped work-type language verbatim: the lucide Hand glyph, --el-type-manual (= --color-stone), and the color-mix(in srgb, var(--el-type-manual) 18%, var(--el-page-bg)) tint — from components/issues/WorkItemTypeChip.tsx / lib/issues/workItemTypeMeta.ts and the ready-list manual design (design/ready/work-type-manual.mock.html). No new hue, no new glyph.
  • One canvas adaptation — a hairline 1px solid --el-border (tinted a touch by the hue) so the chip lifts off EVERY node fill (surface / --el-tint-sky done / --el-tint-mint ready / --el-muted not-in-sprint). The ready-list chip sits on a white row and needs none; this is a variant, not a redraw.
  • Shape via element-semantic tokens (--radius-badge, --spacing-chip-*) — reshapes under data-style; label --el-text-strong for AA on the light stone tint.

Composes with every data-node-state — no collision

The chip lives in the top row and never touches the card fill or border, which is where the whole state-chrome precedence lives (cross-blocked > here > done > ready > not-in-sprint > normal). So it rides ON TOP of every state (sheet 2). Two adjacency rules keep the 280-wide row from crowding: with the quiet not-in-sprint tag the two small chips sit side by side; with the loud red cross-blocked flag the flag wins the row and the Manual chip collapses to its icon (title tooltip retains the label).

Why a chip and not a whole-card wash

The card-fill washes are deliberately reserved for status — done (--el-tint-sky) and ready (--el-tint-mint) — because status is the primary scan axis and a wash survives zoom-out where a pill vanishes (MOTIR-1422). Work type is a secondary, legible-zoom hint (who picks this up), so a type wash would both collide with the status washes and overload the fill. The chip is the honest altitude for it — exactly how type is shown everywhere else in the app.

Access path

None new. The roadmap view (app/(authed)/roadmap/page.tsx) is already a first-class nav entry; this only changes an existing node's appearance — no new door to draw.
