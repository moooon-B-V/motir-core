Direct — populated (default)
Why Watching is disabled

Direct = the actions involving you (mentioned · assigned · reporter — "mentions first"), shipped in 5.7. Watching is drawn disabled (the exact grammar 2.5.3 used for the view-switcher seam) because issue-watching's work-item/transitioned event is Story 5.4's; the 5.7.3 fan-in consumer is built to populate it with NO 5.7 change. A title tooltip explains the wait.

3 · Overflow menu — "Mark all as read" (one server op) + Notification settings
Notifications

"Mark all as read" is a single server operation over the caller's whole unread set (a bulk updateMany), NOT a client loop over the rendered rows — that loop is the JRACLOUD-85017 bug where reload re-shows the dots. After it runs, every dot clears and the count returns 0 from the mutation response (no whole-tree refresh — the inline-edit memory). "Notification settings" deep-links to /settings/account → Notifications (preferences.mock.html). The menu reuses the IssueViewSwitcher overflow vocabulary (--radius-card + --shadow-elevated, --radius-control rows).
