Direct events — mentions, comments on your items, and assignments — default on for both channels. An untouched row uses that default (it is not off until you turn it off).

The section is a Card (the <h2> header grammar LanguageCard / NameCard use) on the existing /settings/account area. Rows = event types; columns = the two channels (Email · In-app); each cell is a Switch. The resolver supplies the documented default for an unset row, so an untouched matrix already reflects "direct events on" without writing a row. The Watching event row is drawn disabled with a "Soon" tag — the Story 5.4 seam (the same seam the drawer's Watching tab draws).

Each toggle is an inline mutation through a Server Action; the success response IS the confirmation — the cell trusts its own success and the switch stays where the user put it, with NO router.refresh() / revalidatePath whole-tree fan-out (the inline-edit-no-tree-refresh memory; the fan-out is what caused the revert bug). A transient "Saved" confirms; on failure the switch reverts to its prior position and an inline "Couldn't save · Retry" appears.

The shipped Switch: role="switch" + aria-checked, a --el-accent track when on, --el-muted + --el-border-strong when off, the --el-surface knob with --shadow-subtle.
