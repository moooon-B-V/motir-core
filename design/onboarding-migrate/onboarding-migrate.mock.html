Migrate-onboarding wizard — link your codebase; everything else is optional

Story 7.15 (MOTIR-815, Workflow B) · design gate 7.15.1 (MOTIR-930). Onboard an EXISTING codebase. A project usually spans several repositories, so the flow is multi-repo. The required core is just Connect + Index — link your repos, Motir reads the code. From the code it silently derives per-repo coding conventions + a code-health check and uses them automatically — nothing to approve, nothing surfaced in onboarding (they live on the Code health page for whoever wants them, 7.14). Then an optional tier: import your existing backlog (Jira / Linear / GitHub / Plane / CSV — composes design/import), then plan in the EXISTING universal plan screen — the PlanWithAILauncher → PlanningWorkspace (MOTIR-1193/1299), which is already implemented, so planning is NOT designed here; after the import step the wizard just opens it. Nothing is forced — after set-up the user imports / plans now or finishes and does it later; Plan with AI is always in the top bar. This card designs the SHELL + index + import-embed and COMPOSES the already-built surfaces (7.7.1 connect · design/import · the universal plan screen). Toggle the theme →

Every step persists its result — including per-repo index status and per-repo convention approval (MOTIR-1499's MigrateOnboarding state machine, MOTIR-931's resumable routes) — so Save & exit and any drop return the user to the exact saved step.
