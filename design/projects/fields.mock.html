The archive / delete split is the verified mirror rule: an in-use option’s Delete is disabled with the tooltip (“In use on N issues — archive instead”); an archived option keeps rendering on issues that hold it and is hidden from new selection; delete is enabled only at zero usage (the unused “Critical” row after its values cleared — and the service re-checks; the DB Restrict backstops). Rename is inline (the “Medium” row, --el-accent focus border + Save). Archived rows lose their grip (not reorderable) and sit last.

5 · Delete field — the confirm names the value count (team-managed hard delete: immediate, permanent, values destroyed). Danger confirm + ghost cancel (the archive-confirm / 2.4.9 grammar).

Delete Severity?

Same heading grammar as the archive-confirm (TriangleAlert in a --el-tint-rose circle), without the typed-identifier arm step — the value count IS the consequence statement, fetched fresh when the confirm opens. A field with no values reads “Deletes the field. No issues hold a value for it.”
