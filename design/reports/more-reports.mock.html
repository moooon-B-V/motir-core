More reports — average age · resolution time · workload — design mock

Story 8.8 · Subtask 8.8.7. The three NEW report pages the Reports-hub “More reports” placeholder always promised — Average age, Resolution time, and Workload — drawn in the grammar of the shipped report pages (Created-vs-Resolved, Status distribution), plus the hub change that turns the dead dashed placeholder into three live, clickable cards. Built from the shipped design system + the 4.6.1 chart language; reuses ReportPageChrome, the panel-4 report-control vocabulary (scope XOR · period · days-back · measure), the 4.6.2 SVG chart layer, the legend + <details> data-table a11y pattern, and the panel-5 widget states. Source of truth for 8.8.13 (the implementation + the widget-registry additions). Mirror product: Jira’s Average Age / Resolution Time / Workload reports (decision-ladder rung 1 — verified against Atlassian docs, not asserted).

Why bars, not Jira’s pie (justified deviation, rung 1). Jira’s Workload report is a pie by assignee, but a pie reads poorly for a ranking across a team-sized set and the shared donut degrades past 7 segments (the “+N more” roll-up). Workload’s job is who is carrying how much — a magnitude RANKING — which horizontal bars sorted descending read directly, with no overflow cliff. Measure = story points (Motir has no time-tracking, so points are the workload unit; Jira’s time-field picker maps to our Story-points ↔ Issue-count toggle). Counts unresolved work only (non-done-category); the unassigned bucket uses the neutral “None” colour and is always last. Current work_item rows grouped by assigneeId — one bounded query, no revision trail needed.

Access path is drawn end-to-end (the design-content rule): the shell Reports nav link → the Issue-analysis card (the door, panel 1) → the report page, and ReportPageChrome’s “Back to reports” crumb reverses it. All three new reports REUSE the shared states verbatim — loading = a skeleton in the chart’s shape; empty = the report-specific zero-data copy (“nothing resolved yet” / “no open work”); error = ErrorState + Retry; no-access = the 6.4 per-viewer gate (a saved-filter scope the viewer can’t see leaks no figures); stale = the inherited 6.2.2 “filter missing” body + Choose a filter; plus the page-level no-project empty state. Every state is carried by TEXT, never colour alone (finding #35).
