Who can manage boards
Board CRUD is a project-config write — membership-gated today (any project member). Under Story 6.4 it becomes project-admin-gated; a non-admin then sees the switcher (to switch boards) with the New / manage affordances hidden, and the server re-gates every write 403.

Loading — the switcher trigger renders as a skeleton while GET /api/boards resolves; the board still shows the current selection. Error — an inline ErrorState with Retry (the 3.2.2 board pattern); a failed board-list never blanks the board. Permissions — membership-gated now, project-admin under 6.4 (no early RBAC).

All colour via --el-*, all shape via element tokens, AA-safe in both themes. State carried by text + icon, never colour alone (finding #35).
