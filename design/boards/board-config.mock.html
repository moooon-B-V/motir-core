Decided at 3.6.2 (mirrors Jira’s “you can’t delete a column with issues”): deleting a column unmaps its statuses (they return to the rail) and removes only the board_column row — never a work item. A column whose mapped status STILL holds board cards is refused (ColumnNotEmptyError → 409); the confirm becomes a guard whose primary action deep-links to remap that status first. State carried by text + icon, never colour alone (finding #35).

Loading: a header + name-field + column skeletons stream in. Error / no-board: an ErrorState with Retry (the 3.2.2 board pattern). All-mapped: the rail shows a positive empty state, not a blank box. (A board always auto-seeds ≥1 column per the Story-3.1 seed — 3.1.2 — so there is no zero-column state; “empty” means a brand-new board whose only column holds the initial status and whose rail is empty.) Save feedback: the per-write “Saving…/Saved” chip on the name field; a failed write surfaces a danger Toast and reverts the optimistic change.
