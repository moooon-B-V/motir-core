4 · Board resolution + permissions — absent ?board= → default · cross-tenant → not-found · membership gate
No ?board= → the project default board. Opening /settings/project/board with no param resolves the project’s isDefault board (3.7.2) and names it in the header — the existing default-only behaviour is preserved, never broken.
A board outside the active project / workspace → not-found. A ?board=<id> that doesn’t belong to the active project (or another tenant’s board) resolves to a 404 ErrorState, never a cross-tenant read — the same tenant-safety guard the board read enforces (3.7.5).
Membership-gated now; project-admin under Story 6.4. Board settings is a board-config write, so (matching the 2.2.5 workflow editor + the 3.3 / 3.6 board config) it is membership-gated today with a TODO(6.4). A non-admin then sees the surface read-only (the 3.6.1 panel-5 read-only treatment) and the server re-gates every write 403, regardless of the rendered affordances.

Resolution + permission states for the per-board settings page — completeness, not the green path only: the default-board fallback keeps the page working with no param; the cross-tenant 404 keeps it tenant-safe; the membership gate matches the sibling board-config surfaces.
