Bounded, never load-all (finding #57). The header count (1,284 issues) is the aggregate from getBacklog, not a loaded-row tally. Only the rows in (or near) the viewport are mounted — the useRowWindow window (3.2.5) keeps the DOM bounded; a spacer preserves scroll height. Scrolling to the bottom trips the sentinel, which fetches the next cursor page and appends. Drag composes with the window: the dragged node stays attached when it scrolls out, and a drop into a not-yet-loaded region resolves to a rank-append (O(1)) — no load-all forced.

While a page is in flight the in-flow spinner + "Loading more…" shows at the bottom; a failed page leaves a focusable inline retry (recoverable without a reload). When the cursor is exhausted the sentinel is replaced by a quiet "All N issues loaded" end-cap — no infinite spinner. The same shape applies to a sprint container bound to getSprintIssues.
