3 · Show instruction → modal: Modal + MarkdownView over descriptionMd

Register the Google OAuth client + secret

PROD-45 Manual Human task · assigned to Alice Chen

A human must create the OAuth client in Google Cloud — a coding agent has no console access. Once the credentials exist, drop them into the env and the dependent code subtask (PROD-42) unblocks.

Steps

  1. Open the Google Cloud Console → APIs & Services → Credentials.
  2. Create an OAuth client ID of type Web application.
  3. Add the authorized redirect URI: https://app.motir.co/api/auth/callback/google.
  4. Copy the Client ID and Client secret.

Then

Set GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in the Vercel project env (Production + Preview), then mark this item done.

Composes the shipped Modal (size lg = 32rem) + MarkdownView rendering the item's descriptionMd — the SAME markdown stack and motir-prose styling as the issue detail page, so the run-instruction reads identically there. Header: the item title (serif) + a subhead with the mono key, the Manual chip, and the assignee. The × and the footer Close both dismiss (Radix handles ESC / click-outside / focus-trap / focus-return).

5 · Long instruction — body scrolls, title + footer pinned (max-h 90vh)

Provision the Vercel Blob store + token

PROD-58 Manual Human task · assigned to Riya Shah

Attachments (subtask 2.3.7) need a Vercel Blob store and a read/write token. This is dashboard + secret work a coding agent can't do.

1 · Create the store

  1. Vercel dashboard → the motir-core project → Storage.
  2. Create a Blob store; name it motir-attachments.
  3. Connect it to Production, Preview, and Development.

2 · Wire the token

Connecting the store auto-adds BLOB_READ_WRITE_TOKEN to the project env. Confirm it appears under Settings → Environment Variables for all three targets.

3 · Local dev

Run vercel env pull .env.local so the token lands in your local env, then restart the dev server.

4 · Verify

  1. Open any work item → Attachments.
  2. Upload a small image; it should appear with a thumbnail.
  3. Reload — the attachment persists (served from Blob).

Then

Mark this item done; the dependent upload subtask (PROD-59) unblocks.

Long descriptionMd scrolls inside Modal.Body (flex-1 overflow-y-auto) while the title, subhead, and footer stay pinned — the panel never exceeds max-h-[90vh]. (Here the stage caps the panel shorter so the scroll is visible in the export.)
